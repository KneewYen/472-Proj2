Difficulties Faced
------------------
1. Malformed dataset header:
   The provided spam.csv had a typo'd header ("lable" instead of "label")
   along with three extra unnamed trailing columns. We resolved this by
   renaming the first two columns positionally after loading the CSV and
   dropping the unused columns instead of relying on hardcoded column
   names.

2. Understanding TF-IDF from scratch:
   None of us had used TfidfVectorizer before. We read the scikit-learn
   documentation and experimented in a small test script to understand
   that TF-IDF converts each message into a vector of weighted word
   importance scores before it reaches the classifier.

3. Low spam recall on the first trained model:
   Our first model (default MultinomialNB with English stop-word removal)
   reached 96.9% accuracy but missed 35 actual spam messages, since the
   dataset is imbalanced (about 87% ham). We fixed this by removing
   stop-word filtering and lowering the Naive Bayes alpha smoothing
   parameter to 0.05, which raised accuracy to 98.4% and significantly
   reduced missed spam messages.
